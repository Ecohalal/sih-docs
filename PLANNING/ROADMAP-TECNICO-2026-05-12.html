Decisões arquiteturais novas

  1. Armazenamento canônico em S3: Certificate.pdfUrl e qrCodeUrl no banco devem armazenar URI s3://bucket/key, NUNCA URLs HTTP públicas. URLs HTTP geradas on-demand via presignS3Uri().
  2. Backward compat sem migration: presignS3Uri() aceita tanto s3://... quanto https://bucket.s3.*.amazonaws.com/... legadas. Sem UPDATE em massa.
  3. PDF imutável, link presigned regenerado: generateAndStore() tem cache forte. Forçar regeneração requer forceRegenerate: true explícito.
  4. TTL 15min (expiresIn: 900): suficiente para download imediato. Re-uso chama o endpoint dedicado.
  5. Role gating de download: GET /:id/download-url aberto a praticamente todas as roles internas. Decisão consciente: certificado é documento oficial. Acesso público via QR Code → /verify/:certNumber (sem download).

Migration confirmada em prod (DBeaver, 2026-05-12 18:13Z)

  • (B) enum CertificateIssuanceMode = {workflow, manual} OK
  • (C) _prisma_migrations: hash real, finished_at preenchido, applied_steps_count=1 OK

7. Quality gates pré-deploy

Checklist obrigatório antes de qualquer push em ecohalal/release:

  1. Migrations — gerar, validar localmente, registrar em prisma/migrations/. Se AUTO_MIGRATE não rodar, aplicar manualmente.
  2. Swagger — npm run generate:swagger + node scripts/generate-api-gateway.js. Commit dos dois arquivos.
  3. TS strict — tsc -b verde (não só tsc --noEmit).
  4. Zod v4 — DTOs com Create/Update separados; .refine() no Refined, não no Base.
  5. Tests — pnpm test verde; aceitar legacy se contado e justificado.
  6. Branch — trabalhar em release direto (sem PR release→main); push dispara CI/CD.
  7. Reconciliação — após hotfix em release, merge release → develop para não perder.
